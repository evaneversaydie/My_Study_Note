# 期末心得
資料結構與演算法課程，幾乎是整個學期花最多時間以及跨出原本圈圈的課了!
期初在實作與紀錄怎麼裝git、github、markdown時，感覺像是從新開始學一樣，從一開始不會用github、沒有寫過markdown到現在可以用github記錄這學期的點點滴滴，真的進步很多。這學期也發現了很多github上的資源，像是文字探勘套件CKIP的官方說明文件、平常練習爬蟲時，github上其他python分享的經驗或API。以及很多網路上的教學文章，這溪都是之前不太會去看的內容。
<br>剛開始從頭摸索，看到很多平常不會用的語法時，一開始對def還很不熟。總會覺得怎麼那麼多看不懂的事情，markdown也花了很多時間才熟練，沒想到現在可以完成很多的筆記。也意外發現了很多的工具。寫作業時也會很崩潰很懷疑自己，很多次熬夜寫功課到凌晨，花了三四十個小時，一邊想著自己真的可以寫出來嗎?(一開始的作業真的常常跟老師一開學、分析班上填寫的表單實說的「抱著電腦看到天已經亮了」的感覺，還會想天亮了我的code的進展呢?)。但經過這學期的訓練後，到期末會發現之前不熟的class、def等等都變的熟悉‧。曾經期初的難題變得沒那麼難了。從leetcode、Codesignal到作業與CS50，很高興自己完成了。所以「要相信這些是可能的」，只要勇敢踏出這一步去做就好了。

<br>
最後，列出印象深刻的事情與老師的提醒班上的觀念，希望之後也不會忘記，懷來翻時也可以受益良多:
 * 不要畫地自限，你可以的!
  * 要相信性格心理學的自我效能啊啊啊!
 * 先了解邏輯再去書寫程式碼。才能了解自己正在做什麼、避免「程式運作了但不知道為什麼可以運作」‧。
 * 切記`「技術債」`。
 * 把學會的東西、做過的東西都成鋪墊的、可以使用工具。可以花更多時間在新學的事情上。像是:寫了Stack和Queque的class，BFS、DFS可以直接引用。
 * 動手做出來與憑空思考之間有一大段距離:即使課堂上畫出流程圖，在實作時也往往遇到困難。
 * 找到可以學習的資源，這些資源比想像的還多:「每個人不論在哪裡，都可以利用這些資願自己去學」。老師推薦leetcode、codesignal、CS50都是網路上的學習資源。要多多去探索。
 * 保持學習的習慣，建立學習的模式:每天都去思考作業怎麼寫，比起花一整天來的好多了。思路也會比較有彈性。保持每天學固定自習的時段是一件很重要的事情。
 * 擅用工具: python 是動態語言，相比C等等對於語法的complier容錯率很大。在寫`HW3`時，發現使用Debug Mode去了解程式如何運行、邏輯上出了什麼錯誤與寫很多個print相比是更快更有效率的方式。
 * 去嘗試外文的網站、非python 教學文，往往比平常查看的中文文章有更多驚喜。
 * 列出學習的軌跡，才能更清楚自己學了什麼、學到哪裡，也方便之後的查詢修改。
 * 學術倫理:參考任何網站都需要留下refernce，包含自我引用。

這學期結束的好快~最後，謝謝助教與老師!!!2020新年快樂!!
